Data Ingestion & Cleaning
Step 1
The pipeline begins by loading the NASA Exoplanet Archive CSV. From ~50+ available columns, 18 scientifically meaningful fields are selected — covering orbital mechanics, stellar properties, and discovery metadata. Missing numeric values are imputed using per-discovery-method group medians rather than global means, preserving the characteristic distributions of each detection technique (e.g. transit-detected planets have different orbital period profiles than radial-velocity ones).

Natural Language Generation
Step 2
Each planet row is converted into a rich, human-readable paragraph using a deterministic template engine — no external LLM required. Three classification helpers first convert raw numerics into qualitative labels (orbit shape, mass category, star type), then a f-string template assembles a coherent paragraph. A habitable-zone hint is conditionally appended when orbital semi-major axis and stellar temperature both fall within Earth-like ranges.
template engine orbit classification mass bucketing stellar typing hab-zone detection

VADER Sentiment Analysis
Step 3 · NLP
A lightweight VADER-style lexicon is applied to each generated description to produce a compound sentiment score in [-1.0, +1.0]. The lexicon is tuned specifically to this domain — words like "habitable" (+2.5), "promising" (+2.0), and "Sun-like" (+1.0) score highly, while "controversial" (−1.5), "unstable" (−0.5), and "cold" (−0.3) score negatively. Scores are normalised with a compression function and bucketed into Positive / Neutral / Negative labels.
VADER-style lexicon compound score [-1, +1] Positive ≥ 0.05 Negative ≤ −0.05 Neutral otherwise
# Domain-specific sentiment lexicon _LEXICON = { "habitable": 2.5, "promising": 2.0, "sun-like": 1.0, "stable": 0.5, "circular": 0.5, "warm": 0.3, "controversial": -1.5, "unstable": -0.5, "cold": -0.3, "dwarf": -0.3, "elliptical": -0.2 } def _vader_score(text): raw = sum(_LEXICON.get(w, 0.0) for w in text.lower().split()) return max(-1.0, min(1.0, raw / (1 + abs(raw) * 0.1)))

Habitability Classification
Step 4 · Rule-based
A 9-point rubric evaluates each planet's suitability for life using 5 astronomically grounded rules. Points are awarded for orbital semi-major axis in the habitable zone, Sun-like stellar temperature, small planetary mass (rocky body), low orbital eccentricity (stable seasons), and stable stellar mass. The raw score maps to three classes used throughout the dashboard.

TF-IDF + K-Means Clustering
Step 5 · ML
The 4,367 generated descriptions are vectorised using TF-IDF (top 500 features, English stop-words removed), then grouped into 6 planet archetypes via K-Means clustering (k-means++, n_init=10, seed=42). Each cluster is labelled by its centroid's highest-weighted TF-IDF term — producing human-readable archetype names like "C0:habitable" or "C3:elliptical". If scikit-learn is unavailable, the step is gracefully skipped.
TF-IDF · 500 features K-Means · k=6 k-means++ init sklearn optional graceful fallback
from sklearn.feature_extraction.text import TfidfVectorizer from sklearn.cluster import KMeans vectorizer = TfidfVectorizer(max_features=500, stop_words="english") X = vectorizer.fit_transform(df["description"]) km = KMeans(n_clusters=6, random_state=42, n_init=10) df["cluster"] = km.fit_predict(X) # Auto-label each cluster from its top TF-IDF term for cid in range(6): top_term = terms[km.cluster_centers_[cid].argsort()[-1]] cluster_labels[cid] = f"C{cid}:{top_term}"
